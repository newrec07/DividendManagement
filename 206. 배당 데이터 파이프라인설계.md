\## PART 1 — 파이프라인 종류 3가지  
<br>
\### Pipeline A — 최초 1회 수집 (Static Bootstrap)  
앱 최초 설치 시 단 1회 실행. 이후 재실행 불필요.  
<br>
\`\`\`  
트리거: 수동 실행 (python bootstrap.py)  
대상: tickers.json에 등록된 전 종목  
소요시간 예상: 약 10~15분 (20종목 × API 호출)  
FMP 250회/일 한도 초과 시 → 2일로 나눠서 실행 (체크포인트 파일로 이어서 진행)  
\`\`\`  
<br>
\*\*수집 순서:\*\*  
\`\`\`  
1\. dividends.json     과거 10년 배당 이력 (특별배당 플래그 포함)  
2\. financials.json    FMP 5년 + SEC EDGAR 6~10년치 보완 (정규화 레이어 적용)  
3\. price\_history.json 과거 5년 주봉 + pandas-ta 기술지표 계산  
4\. valuation\_bands.json 5년 PE밴드 / 배당수익률 밴드 산출  
\`\`\`  
<br>
\*\*실행 방식:\*\*  
\`\`\`python  
\# 실행 커맨드  
python pipeline/bootstrap.py  
<br>
\# 내부 흐름  
for ticker in PENDING\_TICKERS:  # bootstrap\_progress.json의 pending 목록 기준  
    collect\_dividends(ticker)       → /data/static/{ticker}/dividends.json  
    collect\_financials(ticker)      → /data/static/{ticker}/financials.json  
    collect\_price\_history(ticker)   → /data/static/{ticker}/price\_history.json  
    calculate\_valuation\_bands(ticker) → /data/static/{ticker}/valuation\_bands.json  
    mark\_completed(ticker)          → bootstrap\_progress.json 업데이트  
    time.sleep(1)                   # API 레이트 리밋 방지  
\`\`\`  
<br>
\*\*체크포인트 관리:\*\*  
\`\`\`python  
\# /data/bootstrap\_progress.json 기준으로 진행 상태 관리  
\# 재실행 시 completed 목록 스킵 → pending 목록부터 이어서 실행  
\# 파일 불완전 생성(수집 중 크래시) 케이스도 안전하게 처리  
<br>
{  
  "completed": \["ABBV", "HD"\],  
  "pending": \["PLD", "WSO", ...\],  
  "last\_run": "2025-01-10",  
  "status": "IN\_PROGRESS"  // DONE / IN\_PROGRESS / FAILED  
}  
\`\`\`  
<br>
\*\*실패 처리:\*\*  
\`\`\`  
종목별 독립 실행. 1개 실패해도 나머지 계속 진행.  
실패 종목은 /logs/bootstrap\_errors.log 기록.  
재실행 시 bootstrap\_progress.json의 pending 목록부터 자동 이어서 실행.  
\`\`\`

\### Pipeline B — 주간 Live 갱신 (Weekly Refresh)  
매주 금요일 자동 실행. Live 데이터 덮어쓰기 + 점수 재계산.  
<br>
\`\`\`  
트리거: Railway Cron Job — 매주 금요일 17:00 ET  
        (APScheduler 대체. Railway 슬립 모드와 무관하게 안정적 실행)  
대상: 전 종목  
소요시간 예상: 약 5분  
\`\`\`  
<br>
\*\*실행 순서:\*\*  
\`\`\`  
STEP 1. 시장 공통 데이터 수집  
  VIX 현재값 → /data/live/market.json  
<br>
STEP 2. 종목별 Live 스냅샷 수집 (전 종목)  
  현재가 / 배당수익률 / PE  
  52주 고점  
  SMA 13W / 40W / RSI 14W / 볼린저밴드  
  → /data/live/{ticker}/snapshot.json  
<br>
STEP 3. Forward 컨센서스 수집 (전 종목)  
  EPS 컨센서스 Q+1~Q+8  
  EPS 리비전 방향  
  → /data/live/{ticker}/consensus.json  
<br>
STEP 4. 점수 계산 실행  
  Static + Live 데이터 합산  
  201 / 202 / 203 점수 산출  
  → score\_cache\_global 테이블 upsert (공용. 전 사용자 공유)  
<br>
STEP 5. 완료 로그 기록  
  → /logs/weekly\_refresh.log  
\`\`\`  
<br>
\*\*Railway Cron Job 설정:\*\*  
\`\`\`  
\# railway.toml 또는 Railway 대시보드 Cron 설정  
\[cron\]  
schedule = "0 17 \* \* 5"   # 매주 금요일 17:00 ET  
command = "python pipeline/weekly\_refresh.py"  
<br>
\# APScheduler 코드 제거. Railway Cron이 직접 스크립트 실행.  
\# 슬립 모드 진입 여부와 무관하게 Railway 인프라에서 트리거.  
\`\`\`

\## PART 3 — 기술지표 계산 방식  
FMP는 주가 원시값만 제공. 기술지표는 pandas-ta로 직접 계산.  
<br>
\`\`\`python  
\# collectors/price\_history.py 내부 로직  
import pandas as pd  
import pandas\_ta as ta  
import requests  
<br>
def calculate\_indicators(ticker: str):  
    # FMP에서 주봉 데이터 수집 (5년)  
    url = f"https://financialmodelingprep.com/api/v3/historical-price-full/{ticker}"  
    params = {"apikey": FMP\_KEY, "from": "5년전날짜", "to": "오늘"}  
    df = fetch\_and\_resample\_to\_weekly(url, params)  # 일봉 → 주봉 resample('W')  
<br>
    # pandas-ta로 기술지표 계산  
    df\['sma\_13w'\] = ta.sma(df\['Close'\], length=13)  
    df\['sma\_40w'\] = ta.sma(df\['Close'\], length=40)  
<br>
    # 볼린저밴드 (40주 기준)  
    bb = ta.bbands(df\['Close'\], length=40, std=2)  
    df\['bb\_upper'\] = bb\['BBU\_40\_2.0'\]  
    df\['bb\_mid'\]   = bb\['BBM\_40\_2.0'\]  
    df\['bb\_lower'\] = bb\['BBL\_40\_2.0'\]  
<br>
    # RSI (14주)  
    df\['rsi\_14w'\] = ta.rsi(df\['Close'\], length=14)  
<br>
    # 52주 고점 (rolling)  
    df\['high\_52w'\] = df\['High'\].rolling(window=52).max()  
<br>
    return df  
<br>

\## PART 6 — 오류 처리 원칙  
\`\`\`  
API 호출 실패:  
  → 최대 3회 retry (1초 간격)  
  → 3회 모두 실패 시 이전 데이터 유지 + 로그 기록  
  → 프론트에 "데이터 갱신 실패 — 이전 데이터 표시 중" 배너  
<br>
점수 계산 실패 (데이터 누락):  
  → 해당 요소 점수 null 처리  
  → 총점에서 제외 + 프론트에 "일부 지표 계산 불가" 표시  
<br>
FMP 레이트 리밋:  
  → 종목 간 1초 sleep  
  → 배치 실행 시 10종목마다 5초 대기  
  → 250회 한도 초과 시 당일 중단 + bootstrap\_progress.json 업데이트  
     → 다음날 재실행 시 pending 목록부터 자동 이어서 진행  
<br>
EDGAR 파싱 실패:  
  → 종목별 독립 실행. 1개 실패해도 나머지 계속 진행.  
  → 실패 종목 /logs/bootstrap\_errors.log 기록  
  → 수동 보완 후 해당 종목만 On-Demand 재수집
