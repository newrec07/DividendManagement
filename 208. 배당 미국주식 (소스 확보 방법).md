\## 1. 배경 — yfinance 사용 불가 판정  
기존 설계(205/206)에서 yfinance를 메인 소스로 지정했으나, 실제 환경에서 차단 확인. 대안 소스로 FMP 확정.  
<br>
\*\*yfinance 문제점\*\*  
\- 공식 API가 아닌 웹 스크래핑 기반  
\- Yahoo Finance 사이트 변경 시 즉시 중단 위험  
\- IP 차단 및 레이트 리밋 빈번 발생  
\- 프로덕션 환경에서 안정성 보장 불가

\## 2. 데이터 소스 확정  
<br>
\### 메인: FMP (Financial Modeling Prep) 무료 플랜  
<br>
\*\*기본 스펙\*\*  
\- 무료 플랜: 250 API 호출/일  
\- 히스토리: 약 5년  
\- 엔드포인트: 150개 이상  
\- 데이터 원천: SEC EDGAR 직접 소싱 → 정확도 높음  
<br>
\*\*커버 가능 항목\*\*  
| 필요 데이터 | 가능 여부 |  
|------------|----------|  
| 주가 히스토리 (5년 주봉) | ✅ |  
| 배당 이력 (label 필드 특별배당 확인 가능) | ✅ |  
| 손익계산서 / 현금흐름 / 재무상태표 (분기) | ✅ |  
| EPS 컨센서스 / 애널리스트 추정 Q+1~Q+8 | ✅ |  
| 내부자 거래 (insider\_transactions) | ✅ |  
| 뉴스 (기본) | ✅ |  
| VIX (^VIX 티커) | ✅ |  
| 어닝콜 트랜스크립트 | ❌ 무료 플랜 미지원 → Alpha Vantage 뉴스 감성으로 대체 |  
| 10년치 재무 전체 | ⚠️ 5년 초과분 → SEC EDGAR로 보완 |  
<br>
\---  
<br>
\### 보조 1: SEC EDGAR (완전 무료 / 무제한)  
<br>
\*\*용도\*\*: FMP 무료 플랜의 5년 한도를 초과하는 \*\*10년치 재무제표 보완\*\*  
\- 미국 상장사 전체 공시 데이터 무료 제공  
\- FMP 자체도 EDGAR를 원천으로 사용  
\- Bootstrap 1회 실행 시에만 사용 (이후 불필요)  
<br>
\*\*수집 대상\*\*  
\- 손익계산서 10년 (분기)  
\- 현금흐름표 10년 (분기)  
\- 재무상태표 10년 (연간)  
<br>
\*\*정규화 레이어 필수 (PIPE-05 반영)\*\*  
\`\`\`  
EDGAR 원시 XBRL → FMP 표준 컬럼명 매핑 테이블 적용  
단위 통일: 모두 달러(USD) 기준 (EDGAR 천 달러 단위 주의)  
회계연도 기준 통일: 캘린더 기준으로 변환  
접합 검증: 5년차 값이 FMP/EDGAR 두 소스에서 유사한지 cross-check 필수  
\`\`\`  
<br>
\---  
<br>
\### 보조 2: Alpha Vantage 무료키  
<br>
\*\*용도\*\*: 뉴스 감성 분석 + 어닝콜 트랜스크립트 대체  
\- 무료: 25 req/분, 500 req/일  
\- 20종목 규모에서 충분한 여유  
<br>
\*\*커버 항목\*\*  
| 항목 | 용도 |  
|------|------|  
| NEWS\_SENTIMENT | 203 L2-F3 뉴스 톤 분석 (어닝콜 대체) |  
| NEWS\_SENTIMENT | 203 L2-F5 신용등급 뉴스 키워드 |  
| NEWS\_SENTIMENT | 203 L2-F6 규제·정책 리스크 키워드 |  
| 원문 기사 URL | UI에 반드시 노출. 클릭 시 새 탭으로 원문 이동 |  
<br>
\*\*Alpha Vantage 응답 구조 (UI 연동용)\*\*  
\`\`\`json  
{  
  "title": "기사 제목",  
  "url": "원문 링크",  
  "time\_published": "발행일시",  
  "overall\_sentiment\_score": 0.35,  
  "ticker\_sentiment": \[  
    {  
      "ticker": "ABBV",  
      "ticker\_sentiment\_label": "Bullish"  
    }  
  \]  
}  
\`\`\`  
※ url 필드 반드시 UI에 노출. 감성 점수만 표시하고 원문 링크 누락 금지.

\## 4. API 호출 수 계산 (20종목 기준)  
<br>
\### Bootstrap (최초 1회)  
| 엔드포인트 | 종목당 호출 | 20종목 합계 |  
|-----------|-----------|-----------|  
| 주가 히스토리 | 1회 | 20회 |  
| 배당 이력 | 1회 | 20회 |  
| 손익계산서 | 1회 | 20회 |  
| 현금흐름표 | 1회 | 20회 |  
| 재무상태표 | 1회 | 20회 |  
| \*\*합계\*\* | \*\*5회\*\* | \*\*100회\*\* |  
<br>
→ 250 한도 내 1일 완료 가능. 여유 150회.  
→ 한도 초과 시 bootstrap\_progress.json 체크포인트 기준으로 다음날 이어서 실행.  
<br>
\### Weekly Refresh (매주 금요일)  
| 엔드포인트 | 종목당 호출 | 20종목 합계 |  
|-----------|-----------|-----------|  
| 현재가 / 배당수익률 / PER | 1회 | 20회 |  
| EPS 컨센서스 + 리비전 | 1회 | 20회 |  
| 뉴스 | 1회 | 20회 |  
| VIX (공통) | — | 1회 |  
| \*\*합계\*\* | \*\*3회\*\* | \*\*61회\*\* |  
<br>
→ 250 한도 대비 여유 충분. 종목 50개까지 확장 가능.

\## 5. 수집 파이프라인 3단계  
<br>
\### Pipeline A — Bootstrap (최초 1회, 수동 실행)  
\`\`\`  
python bootstrap.py 실행  
↓  
bootstrap\_progress.json 확인 → pending 목록 기준으로 실행  
↓  
FMP → 주가/배당/재무 5년 수집  
↓  
SEC EDGAR → 재무 6~10년치 보완 (정규화 레이어 적용)  
↓  
/data/static/{TICKER}/ 저장  
↓  
완료 종목 → bootstrap\_progress.json completed 업데이트  
실패 종목 → /logs/bootstrap\_errors.log 기록  
↓  
FMP 250회 한도 초과 시 → 당일 중단, 다음날 재실행 시 pending부터 이어서 진행  
\`\`\`  
<br>
\### Pipeline B — Weekly Refresh (매주 금요일 자동)  
\`\`\`  
Railway Cron Job → 매주 금요일 17:00 ET 자동 실행  
(슬립 모드와 무관하게 Railway 인프라에서 직접 트리거)  
↓  
VIX 수집 → market.json 덮어쓰기  
↓  
전 종목 Live 스냅샷 수집 → snapshot.json 덮어쓰기  
↓  
EPS 컨센서스 수집 → consensus.json 덮어쓰기  
↓  
점수 엔진 실행 (201/202/203)  
↓  
score\_cache\_global upsert (공용 테이블)  
↓  
완료 로그 기록  
\`\`\`  
<br>
\### Pipeline C — On-demand (종목 추가 / 수동 갱신)  
\`\`\`  
UI에서 티커 입력  
↓  
FMP로 종목 유효성 확인  
↓  
해당 종목만 Bootstrap 로직 단독 실행  
↓  
Static JSON 생성 완료  
↓  
tickers.json 자동 등록  
↓  
다음 Weekly Refresh부터 자동 포함  
\`\`\`  
<br>

\## 6. 종목 관리 정책  
\- 기본 제공 종목: 사전 등록 10개 (205 문서 참조)  
\- 추가 종목: 앱 내 "종목 추가" 기능으로 티커 입력 → On-demand 자동 수집  
\- 종목 수 제한: 없음. API 호출 한도 기준으로 50개까지 무료 범위 내 운영 가능  
\- 종목 삭제: tickers.json에서 제거 + Static 파일 수동 삭제  
\- 신규 종목 등록 시 \*\*sector\_de\_ratio / wacc 수동 입력 필수\*\* (Bootstrap 전 tickers.json에 미리 입력)

\## 7. 예외 처리 원칙  
\`\`\`  
API 호출 실패:  
  → 최대 3회 retry (1초 간격)  
  → 3회 실패 시 이전 데이터 유지 + 로그 기록  
  → 프론트에 "데이터 갱신 실패 — 이전 데이터 표시 중" 배너  
<br>
데이터 누락 (특정 필드 없음):  
  → 해당 요소 점수 null 처리  
  → 총점에서 제외 + "일부 지표 계산 불가" 표시  
<br>
FMP 무료 한도 초과:  
  → 당일 수집 중단 + bootstrap\_progress.json 업데이트  
  → 다음날 자동 재시도 (pending 목록부터)  
<br>
EDGAR 파싱 실패:  
  → 종목별 독립 실행. 1개 실패해도 나머지 계속 진행.  
  → 실패 종목 /logs/bootstrap\_errors.log 기록  
  → 수동 보완 후 해당 종목만 On-Demand 재수집  
<br>
어닝콜 트랜스크립트 수집 불가:  
  → Alpha Vantage NEWS\_SENTIMENT로 대체 확정  
  → L2-F3 정확도 저하 트레이드오프 인지. UI에 "뉴스 감성 기반 근사값" 표시  
\`\`\`
