\## PART 1 — 모듈 간 트리거 조건 정의  
<br>
\### 201 → 202 관계  
\`\`\`  
병렬 표시. 트리거 없음.  
두 점수는 항상 동시에 계산되어 화면에 함께 표시.  
<br>
신뢰도 강화 조건:  
  201 >= 80 AND 202 >= 80 → "HIGH CONFIDENCE" 배지 표시  
  201 >= 80 AND 202 < 80  → "재검토 필요" 경고  
  201 < 60                → 매수 신호 없음. 203/204 진입 비활성화  
<br>
C2\_BLOCKED 상태:  
  signal\_201 = 'C2\_BLOCKED' 이면  
  → 총점과 무관하게 "⚠️ C2 이익 체력 0점 — 매수 보류" 배지 표시  
  → 203 진입 시 경고 배너 함께 표시  
\`\`\`  
<br>
\### 201/202 → 203 트리거  
\`\`\`  
IF 201\_total >= 60 OR 202\_total >= 80  
  → 203 섹션 활성화 (접근 가능)  
ELSE  
  → 203 섹션 잠금  
  → UI: "매수 신호 미달 — 지속가능성 검증 불필요"  
<br>
C2 필터 추가 조건:  
  IF signal\_201 = 'C2\_BLOCKED'  
    → 203 진입은 가능하나 상단에 경고 배너 고정 표시  
    → "이익 체력 지표 0점 — 펀더멘털 재확인 필요"  
<br>
Yellow Flag 표시:  
  IF yellow\_flag\_count >= 1  
    → 203 섹션 상단에 "⚠️ Yellow Flag {n}개 — 모니터링 강화" 표시  
    → 진입 차단은 아님. 사용자 판단 유도  
\`\`\`  
<br>
\### 203 → 204 트리거  
\`\`\`  
IF grade\_203 = 'STRONG' AND red\_flag\_count = 0  
  → 204 시뮬레이션 활성화  
<br>
IF grade\_203 = 'CAUTION' (60~79점)  
  → 204 버튼 비활성화  
  → UI: "다음 분기 실적 확인 후 재판정"  
<br>
IF grade\_203 = 'DANGER' (60점 미만)  
  → 204 완전 잠금  
  → UI: "배당 안전성 위험 — 시뮬레이션 불가"  
<br>
Red Flag 조건:  
  IF red\_flag\_count >= 1  
    → 203 점수와 무관하게 204 진입 차단  
    → UI: "Red Flag {n}개 감지 — 즉시 재검증 필요"  
<br>
Red Flag 이력 표시:  
  IF red\_flag\_history 존재 (이전에 Red Flag 발생 후 해제된 경우)  
    → 204 활성화되더라도 "이전 Red Flag 이력 있음 (해제일: YYYY-MM-DD)" 표시  
    → 사용자가 직접 확인 후 판단하도록 유도  
\`\`\`  
<br>
\### 204 → 포트폴리오 트리거  
\`\`\`  
시뮬레이션 화면에서 "매수 기록" 버튼 클릭  
→ 포트폴리오 매수 입력 폼으로 이동  
→ 시뮬레이션에서 사용한 매수가 / 수량 자동 채워짐 (수정 가능)  
\`\`\`

\## PART 2 — 화면별 데이터 흐름  
<br>
\### Screen 0 — 로그인 / 회원가입  
\`\`\`  
진입 시:  
  미로그인 상태 → 자동으로 로그인 화면으로 리다이렉트  
<br>
기능:  
  이메일 + 비밀번호 입력  
  로그인 버튼 → JWT 토큰 발급 → localStorage 저장  
  회원가입 버튼  
  에러 메시지 (이메일 중복, 비밀번호 오류 등)  
<br>
토큰 만료 시:  
  → 자동 로그아웃  
  → 로그인 화면으로 리다이렉트  
\`\`\`  
<br>
\### Screen 1 — 대시보드  
\`\`\`  
진입 시 호출:  
  GET /api/scores/all  (JWT 토큰 포함)  
  → score\_cache\_global 전 종목 반환  
<br>
표시 항목:  
  종목명 / 티커  
  201 점수 + 신호 (색상 구분)  
  202 점수 + 신호  
  203 등급 (STRONG / CAUTION / DANGER / 미계산)  
  C2\_BLOCKED 배지 (해당 시)  
  Yellow Flag 개수  
  Red Flag 개수  
  마지막 갱신일  
<br>
정렬 기본값: 201 점수 내림차순  
<br>
필터 버튼:  
  전체 / Strong Buy / Watch / 보유중 / C2\_BLOCKED  
\`\`\`  
<br>
\### Screen 2 — 종목 상세  
\`\`\`  
진입 시 호출:  
  GET /api/scores/{ticker}      점수 상세 + c2\_blocked 상태  
  GET /data/live/{ticker}/snapshot.json  현재 시장 데이터  
  GET /api/news/{ticker}        뉴스 감성 + 원문 링크  
<br>
표시 블록 A — 현재 시장 상태:  
  현재가 / 배당수익률 / PER  
  52주 고점 대비 하락률 (MDD)  
  VIX 현재값  
<br>
표시 블록 B — 201 정밀형 점수:  
  카테고리별 점수 breakdown  
  C1(30) / C2(30) / C3(20) / C4(20)  
  요소별 점수 + 판단 근거 한 줄 텍스트  
  C2\_BLOCKED 시 경고 배너 표시  
  F1/F2 밴드 미형성 시 "밴드 데이터 부족" 표시  
<br>
표시 블록 C — 202 심플형 점수:  
  동일 구조  
  F1 밴드 미형성 시 "밴드 데이터 부족 — 201 정밀형으로 재검토 권고" 표시  
<br>
표시 블록 D — 203 지속가능성:  
  LAYER1 / LAYER2 점수  
  Yellow Flag 목록 (있을 경우)  
  Red Flag 목록 (있을 경우)  
  Red Flag 이력 (해제된 경우 포함)  
  등급 배지 (STRONG / CAUTION / DANGER)  
<br>
표시 블록 E — 뉴스:  
  감성 점수 + 기사 제목 + 원문 링크 (새 탭)  
  Alpha Vantage ticker\_sentiment\_label (Bullish / Neutral / Bearish)  
<br>
하단 액션 버튼:  
  \[시뮬레이션 보기\]  
    → 203 STRONG + red\_flag\_count = 0 일 때만 활성  
    → Red Flag 이력 있으면 활성이나 이력 경고 표시  
  \[매수 기록하기\] → 항상 활성 (점수 무관하게 기록은 가능)  
\`\`\`  
<br>
\### Screen 3 — 시뮬레이션 (204)  
\`\`\`  
진입 시 입력값:  
  매수가 (기본값: snapshot.current\_price 자동 입력)  
  보유 수량  
  시나리오 선택 (Base / Bull / Bear)  
<br>
호출:  
  POST /api/simulation/{ticker}  
  body: { purchase\_price, shares, scenario }  
<br>
표시 항목:  
  출력 #1 분기별 예상 DPS 테이블  
    단기 초록 (Q+1~Q+4, 컨센서스 고정)  
    중기 노랑 (Q+5~Q+12, 블렌딩)  
    장기 회색 (Q+13~Q+40, DGR 리니어)  
    상단 안내: "단기 구간(Q+1~Q+4)은 애널리스트 컨센서스 기반으로 고정됩니다"  
<br>
  출력 #2 누적 배당 곡선 (Recharts)  
    Bull / Base / Bear 3개 라인  
    원금 회수 시점 표시  
<br>
  출력 #3 YOC 추적 차트  
    현재 배당수익률 기준선  
    사용자 설정 목표 YOC 선  
<br>
  출력 #4 목표 매수가 역산  
    원하는 YOC 입력 → 최대 매수가 계산  
<br>
  출력 #5 신호 요약 배너  
    컨센서스 방향 / 뉴스 톤 / 충돌 감지 여부 / 신뢰도  
<br>
시나리오 전환:  
  Base / Bull / Bear 탭 클릭 시  
  → 단기(Q+1~Q+4): 서버 데이터 고정. 변경 없음.  
  → 중기/장기: 프론트에서 DGR값만 변경하여 즉시 재계산 (서버 재호출 없음)  
\`\`\`  
<br>
\### Screen 4 — 포트폴리오  
\`\`\`  
진입 시 호출:  
  GET /api/portfolio/summary  (JWT 포함, user\_id 기준)  
<br>
표시 블록 A — 전체 요약:  
  총 투자금액  
  현재 평가금액  
  미실현 손익 (%)  
  연간 예상 배당금  
  포트폴리오 YOC  
<br>
표시 블록 B — 종목별 보유 현황:  
  티커 / 보유수량 / 평균단가  
  현재가 / 수익률  
  예상 연간 배당금 / YOC  
  Red Flag 보유 종목 → 경고 아이콘 + "즉시 재검증 필요" 툴팁  
<br>
표시 블록 C — 배당 수령 이력:  
  월별 수령 배당금 바차트 (Recharts)  
  종목별 배당 수령 내역  
<br>
액션:  
  \[매수 기록\] 버튼 → 종목 / 날짜 / 수량 / 단가 입력 폼  
  \[매도 기록\] 버튼 → 동일  
  \[배당 수령 기록\] 버튼 → 종목 / 날짜 / DPS / 수량 입력  
\`\`\`

\### 데이터 갱신 트리거  
\`\`\`  
앱 최초 진입:  
  → 토큰 확인 → 없으면 로그인 화면  
  → GET /api/scores/all 호출  
  → allScores 상태 채움  
<br>
종목 클릭:  
  → GET /api/scores/{ticker} 호출  
  → snapshot.json 로드  
  → GET /api/news/{ticker} 호출  
  → tickerDetail 상태 채움  
<br>
시뮬레이션 진입:  
  → purchasePrice 기본값 = snapshot.current\_price  
  → POST /api/simulation/{ticker} 호출  
  → shortTermDPS 별도 저장 (시나리오 전환 시 고정)  
<br>
시나리오 전환 (탭 클릭):  
  → shortTermDPS 유지 (서버 재호출 없음)  
  → 중기/장기만 DGR 변경하여 프론트 즉시 재계산  
<br>
포트폴리오 진입:  
  → GET /api/portfolio/summary 호출 (JWT 포함)  
<br>
수동 갱신 버튼:  
  → POST /api/refresh/all 호출  
  → 완료 후 allScores 재호출

\## PART 4 — Red Flag 전파 로직  
Red Flag는 발생 즉시 UI 전체에 전파.  
<br>
\`\`\`  
Red Flag 발생 (203 계산 시):  
│  
├─ score\_cache\_global에 red\_flag\_count 업데이트  
│  red\_flag\_history에 발생일 기록  
│  
├─ 대시보드: 해당 종목 행에 🔴 배지 표시  
│  
├─ 종목 상세: 상단 경고 배너 고정 표시  
│  "Red Flag {n}개 — {요소명} 이상 감지"  
│  
├─ 시뮬레이션: 진입 차단  
│  
└─ 포트폴리오: 보유 종목일 경우  
   → 보유 현황 행에 경고 아이콘  
   → "즉시 재검증 필요" 툴팁  
<br>
Red Flag 해제 조건:  
  주간 갱신 2회 연속(2주) red\_flag\_count = 0 유지 시 해제  
  → red\_flag\_history에 해제일 기록 (이력 영구 보존)  
  → UI에 "이전 Red Flag 이력 있음 (해제일: YYYY-MM-DD)" 표시  
  → 204 시뮬레이션 재활성화되나 이력 경고는 계속 표시  
\`\`\`
