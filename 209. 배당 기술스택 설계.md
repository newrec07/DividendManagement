\## 1. 확정된 방향 요약  
| 항목 | 결정 |  
|------|------|  
| 실행 환경 | 웹앱 (PWA 지원) — 모바일/데스크탑 동시 대응 |  
| 사용자 수 | 최대 동시 10명 |  
| 인증 | 개인 로그인 — 포트폴리오 + 분석 데이터 모두 사용자별 저장 |

| 기술지표 계산 | pandas-ta |

| 점수 캐시 | score\_cache\_global 공용 테이블. 사용자별 중복 계산 없음 |  
| 스케줄러 | Railway Cron Job (APScheduler 대체. 슬립 모드 방지) |

<br>
\## 2. 전체 기술 스택  
<br>
\### 백엔드  
| 항목 | 선택 | 이유 |  
|------|------|------|  
| 언어 | Python 3.11 | 생태계, 안정성 |  
| API 서버 | FastAPI + uvicorn | 비동기, 경량, 자동 문서화 |  
| 인증 | FastAPI-Users | 회원가입/로그인/JWT 토큰 one-stop 처리 |  
| 스케줄러 | Railway Cron Job | APScheduler 대체. 슬립 모드와 무관하게 안정적 실행 |  
| ORM | SQLAlchemy 2.0 | FastAPI-Users 표준 연동 |  
| 데이터 처리 | pandas + numpy | 핵심 |  
| 기술지표 | pandas-ta | RSI/SMA/볼린저밴드 한 줄 처리 |  
| HTTP 클라이언트 | requests | FMP/Alpha Vantage API 호출 |  
| EDGAR 수집 | sec-edgar-downloader | 10년치 재무 1회 수집 |  
<br>
\### 데이터베이스  
| 항목 | 선택 | 이유 |  
|------|------|------|  
| DB | PostgreSQL | 다중 사용자, 클라우드 배포 필수 |  
| 호스팅 | Supabase 무료티어 | PostgreSQL 무료 제공, 인증 연동 용이 |  
<br>
\*\*Supabase 무료티어 스펙\*\*  
\- PostgreSQL 500MB  
\- 월 200만 API 요청  
\- 10명 동시 사용자 기준 충분  
<br>
\*\*테이블 구분\*\*  
\`\`\`  
공용 테이블 (전 사용자 공유):  
  score\_cache\_global  → 종목별 점수. 사용자 무관 1건만 유지.  
<br>
사용자별 테이블:  
  users               → FastAPI-Users 자동 생성  
  positions           → 매수/매도 이력 (user\_id 포함)  
  dividend\_received   → 배당 수령 이력 (user\_id 포함)  
  watchlist           → 관심 종목 (user\_id 포함)  
<br>
Static JSON 파일은 유지:  
  과거 재무/주가 데이터 → 서버 파일시스템 (/data/static/, /data/live/)  
\`\`\`  
<br>
\### 프론트엔드  
| 항목 | 선택 | 이유 |  
|------|------|------|  
| 프레임워크 | React 18 + Vite | 빠른 빌드, 현재 표준 |  
| 스타일 | Tailwind CSS 3 | 기존 설계 확정 |  
| 상태관리 | Zustand | 가볍고 단순, 사용자별 상태 분리 용이 |  
| 차트 | Recharts | 기존 설계 확정 |  
| 테이블 | TanStack Table | 정렬/필터 내장, 대시보드 필수 |  
| UI 컴포넌트 | shadcn/ui + lucide-react | Tailwind 기반, 모바일 친화적 |  
| PWA | Vite PWA Plugin | 홈 화면 추가, 오프라인 캐시 |  
<br>
\### PWA 구성  
\`\`\`  
vite-plugin-pwa 설정  
├── manifest.json   앱 이름 / 아이콘 / 테마컬러  
├── Service Worker  오프라인 캐시 (정적 자산)  
└── 홈 화면 추가    iOS Safari "홈 화면에 추가" 지원  
\`\`\`

\## 3. 인증 및 사용자별 데이터 구조  
<br>
\### 인증 흐름  
\`\`\`  
회원가입 / 로그인  
↓  
JWT 토큰 발급  
↓  
모든 API 요청에 토큰 포함  
↓  
사용자별 데이터 분리 조회 (user\_id 기준)  
\`\`\`  
<br>
\### DB 테이블 구조  
\`\`\`sql  
\-- 사용자 테이블 (FastAPI-Users 자동 생성)  
users (id, email, hashed\_password, is\_active)  
<br>
\-- 점수 캐시 (공용 — user\_id 없음)  
score\_cache\_global (  
    ticker, snapshot\_date,  
    score\_201\_total, score\_201\_f1 ~ f10,  
    signal\_201, c2\_blocked,  
    score\_202\_total, signal\_202,  
    score\_203\_total, red\_flag\_count, yellow\_flag\_count,  
    grade\_203, red\_flag\_history,  
    updated\_at  
)  
<br>
\-- 포트폴리오 (사용자별)  
positions (id, user\_id, ticker, type, date, shares, price, note)  
dividend\_received (id, user\_id, ticker, pay\_date, dps, shares, total)  
<br>
\-- 관심 종목 (사용자별)  
watchlist (id, user\_id, ticker, added\_date)  
\`\`\`

\## 4. 뉴스 감성 분석 + 원문 링크  
<br>
Alpha Vantage NEWS\_SENTIMENT API 응답에 기사 URL 포함.  
<br>
\`\`\`  
Alpha Vantage 응답 구조  
├── title                    기사 제목  
├── url                      원문 링크 ← UI에 반드시 노출  
├── time\_published  
├── overall\_sentiment\_score  
└── ticker\_sentiment  
    └── ticker\_sentiment\_label  (Bullish / Neutral / Bearish)  
\`\`\`  
<br>
UI에서는 감성 점수와 함께 원문 링크를 카드 형태로 노출.  
클릭 시 새 탭으로 원문 이동.  
감성 점수만 표시하고 원문 링크 누락 금지.

\## 5. 배포 구성  
<br>
\### 무료 클라우드 조합  
\`\`\`  
\[FastAPI 서버\]  
Railway 무료티어  
  └── 월 500시간 실행  
  └── Python 환경 지원  
  └── 환경변수 관리 (API 키 보관)  
  └── Cron Job 기능으로 Weekly Refresh 자동 실행  
<br>
\[React 프론트\]  
Vercel 무료티어  
  └── 정적 빌드 배포  
  └── 글로벌 CDN  
  └── 커스텀 도메인 지원  
<br>
\[PostgreSQL\]  
Supabase 무료티어  
  └── 500MB  
  └── FastAPI-Users 연동  
  └── score\_cache\_global (공용) + 사용자별 테이블  
<br>
\[Static JSON 파일\]  
Railway 서버 파일시스템  
  └── /data/static/, /data/live/ 그대로 유지
